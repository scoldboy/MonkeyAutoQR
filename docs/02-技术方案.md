# 技术方案：选区触发扫描网页图片二维码

## 技术选型

### 运行环境

- **宿主**：Tampermonkey / Violentmonkey 浏览器扩展
- **目标浏览器**：Chrome 80+、Edge 80+、Firefox 78+
- **脚本类型**：UserScript（`.user.js`），用户手动安装

### 二维码识别库：jsQR

- **名称**：[jsQR](https://github.com/cozmo/jsQR)（v1.4.0）
- **选型理由**：
  - 纯 JavaScript 实现，无需 WebAssembly 或原生依赖
  - 可直接处理 Canvas 提取的 `ImageData`
  - 通过 CDN 引入，不增加脚本文件体积
  - 支持多种 QR 码格式（URL、文本、WiFi、vCard 等）
- **引入方式**：`@require https://cdn.jsdelivr.net/npm/jsqr@1.4.0/dist/jsQR.min.js`

### 不引入额外 UI 库

所有 UI 通过原生 DOM + CSS 构建，零依赖。

## 架构设计

```

│                  qr-scanner.user.js               │

│  2. CSS 样式注入                                  │
│  3. 选区按钮管理器 (SelectionButtonManager)        │
│     ├── mouseup 事件监听 + Selection API           │
│     ├── 选区-图片语义包含检测 (containsNode)        │

│     └── 尺寸过滤 + 视口边界检测                    │
│  4. 二维码扫描器 (QRScanner)                       │
│     ├── 图片 → Canvas → ImageData 转换            │

│     └── 跨域图片降级处理                           │
│  5. 结果弹框 (ResultModal)                        │

│     └── 关闭逻辑（ESC / 遮罩点击）                 │
└─────────────────────────────────────────────────┘
```

## 核心数据流

```
用户拖动鼠标选中页面区域
        │
        ▼
  mouseup 事件触发 → 读取 window.getSelection()
        │
        ▼
  图片是否被「选入」选区？(Selection.containsNode)
   —— 仅当图片节点本身包含在选区中才视为选中
   —— 选中相邻纯文字不会包含图片节点，不会误触发
        │
        ├── 否 ──→ 不处理
        ▼ 是
  图片尺寸 ≥ 64×64 ?  ──── 否 ──→ 跳过该图片
        │ 是
        ▼
  为每张符合条件的图片创建按钮，定位右上角
        │
        ▼
  用户点击某张图片的扫描按钮（其他按钮隐藏）
        │
        ▼
  图片 → Canvas → ctx.getImageData()
        │
        ▼
  jsQR(imageData, width, height)
        │
   ┌────┴────┐
   │         │
 有结果    无结果
   │         │
   ▼         ▼
显示结果   显示提示
 弹框       "未检测到二维码"
   │
   ├── 复制 → navigator.clipboard.writeText()
   └── 打开 → window.open(url, '_blank')
        │
        ▼
  用户关闭弹框（ESC / 遮罩 / X按钮）
```

## 跨域图片处理策略

```
图片能否直接绘制到 Canvas？
        │
   ┌────┴────┐
   │         │
 同源/    跨域且无
 CORS头   CORS支持
   │         │
   ▼         ▼
直接绘制  Canvas被"污染"
   │     (SecurityError)
   │         │
   │         ▼
   │  尝试 GM_xmlhttpRequest
   │  以 blob 方式获取图片
   │         │
   │    ┌────┴────┐
   │    │         │
   │  成功获取   请求失败
   │    │         │
   │    ▼         ▼
   │  创建      显示提示
   │  ObjectURL  "跨域限制"
   │  加载后     + 建议右键
   │  绘制Canvas  另存图片后
   │    │         手动扫描
   └────┴──────────┘
        │
        ▼
  ImageData → jsQR
```

## 关键边界情况

| 场景 | 处理方式 |
|------|----------|
| 图片 < 64×64 px | 不显示扫描按钮 |
| 图片是 SVG | 不触发（SVG 无法直接绘制到 Canvas） |
| 图片被 CSS 隐藏（`display:none`） | `offsetParent === null` 检查，不触发 |
| 按钮超出视口右边界 | 改为图片左上角定位 |
| 按钮超出视口下边界 | 向上偏移 |
| 选区覆盖多张图片 | 为每张符合条件的图片分别创建按钮 |
| 点击某张图片的按钮 | 隐藏其他所有按钮，仅保留当前操作的按钮 |
| 点击页面空白处 | 隐藏所有已显示的按钮 |
| 同时多张图片 | 支持多个按钮同时存在 |
| 仅选中图片**旁边**的纯文字 | 图片节点未被选入选区，**不**弹出按钮（v1.1.1 修复） |
| 拖动经过图片形成选区 | 图片节点被纳入选区，正常弹出按钮 |
| 弹框打开时再次点击扫描 | 先关闭旧弹框再显示新弹框 |
| 复制 API 不可用（非 HTTPS） | 降级为 `document.execCommand('copy')` |
| jsQR 返回多个二维码 | 取第一个结果 |
| 页面销毁 / 切换 | 事件监听自动随页面清理 |

## 性能考虑

- 按钮为纯 CSS DOM，创建成本极低（< 1ms）
- Canvas 绘制仅在用户主动点击时触发，不会无故消耗资源
- jsQR 同步执行，对于常规二维码图片（< 500×500 px），识别时间 < 200ms
- 事件监听使用事件委托，仅绑定 `document` 级别，不对每张图片单独绑定
- 选区检测在 mouseup 时执行一次，无持续性能开销
