# 测试用例

## 测试环境

- 浏览器：Chrome / Edge（最新版）
- 扩展：Tampermonkey 或 Violentmonkey
- 脚本：`qr-scanner.user.js`（开发中的最新版本）

---

## 一、选区触发按钮测试

| 编号 | 测试场景 | 预期结果 | 状态 |
|------|----------|----------|------|
| T1.1 | 拖动选中包含 ≥ 64×64 图片的区域 | 图片右上角出现扫描按钮 | ⬜ |
| T1.2 | 拖动选中不包含任何图片的区域 | 不出现按钮 | ⬜ |
| T1.3 | 拖动选中区域覆盖了 < 64×64 的小图 | 小图不显示按钮（过滤） | ⬜ |
| T1.4 | 拖动选中覆盖了 SVG 图片 | SVG 图片不显示按钮 | ⬜ |
| T1.5 | 按钮靠近视口右边缘 | 按钮自动调整到图片左上角，不溢出屏幕 | ⬜ |
| T1.6 | 按钮靠近视口下边缘 | 按钮向上偏移，不溢出屏幕 | ⬜ |
| T1.7 | 拖动选中覆盖多张符合条件的图片 | 每张图片各自显示扫描按钮 | ⬜ |
| T1.8 | 点击页面空白处 | 所有按钮消失 | ⬜ |
| T1.9 | 先点一张图的按钮，再点另一张图的按钮 | 旧按钮隐藏，仅新图片按钮保持显示 | ⬜ |
| T1.10 | 鼠标单纯悬浮图片（不拖动选中） | 不出现按钮 | ⬜ |
| T1.11 | 图片被 CSS `display:none` 隐藏 | 不出现按钮 | ⬜ |
