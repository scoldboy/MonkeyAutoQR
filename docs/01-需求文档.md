# 需求文档：选区触发扫描网页图片二维码

## 项目概述

一款浏览器油猴脚本（Tampermonkey / Violentmonkey），在用户浏览网页时，鼠标拖动选中图片区域后出现扫描按钮，点击后识别图片中的二维码，弹框展示链接地址。

## 目标用户

- 普通网页浏览者，看到二维码图片时想快速获取链接而不需要掏出手机扫描
- 不懂代码的小白用户，需要开箱即用的体验

## 用户故事

1. **选区触发**：作为用户，当我在网页中拖动鼠标选中包含图片的区域时，我希望被选中的图片旁边出现扫描按钮，这样我就能一键扫描。
2. **扫描识别**：作为用户，点击扫描按钮后，我希望脚本自动识别图片中的二维码，并告诉我里面的链接是什么。
3. **查看结果**：作为用户，扫描成功后，我希望弹出一个干净的窗口展示链接，并且能看到复制和打开按钮。
4. **复制链接**：作为用户，我希望点击复制按钮后，链接就复制到了剪贴板，同时有「已复制」的提示。
5. **打开链接**：作为用户，我希望点击打开按钮后，链接会在浏览器新标签页中打开。
6. **非二维码提示**：作为用户，如果一张图片不是二维码，扫描后我希望看到友好的提示，而不是莫名其妙没反应。
7. **跨域提示**：作为用户，如果某张图片因技术限制无法扫描，我希望看到清晰的说明。

## 非功能需求

- **兼容性**：支持 Chrome / Edge / Firefox 上的 Tampermonkey 和 Violentmonkey
- **性能**：选区释放后按钮在 100ms 内出现，扫描在 1 秒内完成
- **UI**：淡蓝色主调，简洁现代风格
- **体积**：脚本主体控制在 500 行以内，不含 jsQR 库

## 不做的功能（范围边界）

- 不在鼠标悬浮时自动显示按钮（改为选区触发，避免误触发）
- 不自动扫描页面上所有图片（避免性能问题）
- 不处理 SVG 格式图片
- 不支持动态图（GIF）中的二维码
- 不保存扫描历史记录
- 不支持摄像头实时扫描
