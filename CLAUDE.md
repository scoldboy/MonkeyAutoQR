# CLAUDE.md — 项目工作指引

## 项目简介

为小白用户开发一款油猴脚本，鼠标悬浮网页图片时出现扫描按钮，点击识别二维码，弹框显示链接。

## 标准文档路径

在开始任何工作前，先读取对应文档：

| 文档 | 路径 | 何时阅读 |
|------|------|----------|
| 需求文档 | `docs/01-需求文档.md` | 新增功能前、不确定需求时 |
| 技术方案 | `docs/02-技术方案.md` | 了解架构、数据流、边界处理 |
| UI 设计规范 | `docs/03-UI设计规范.md` | 修改样式、创建 UI 组件时 |
| 开发步骤 | `docs/04-开发步骤.md` | 了解当前阶段和下一步任务 |
| 测试用例 | `docs/05-测试用例.md` | 阶段 4 测试验证时 |

## 工作约定

### 1. 分阶段推进

- 严格按 `docs/04-开发步骤.md` 中的 5 个阶段顺序执行
- 每个阶段完成后**暂停**，等待用户确认再进入下一阶段
- 不得一个阶段内塞入后续阶段的工作

### 2. 每日日志

- 每次开发会话在 `devlog/` 下创建 `YYYY-MM-DD.md`
- 日志格式：
  ```markdown
  # 开发日志 - YYYY-MM-DD

  ## 今日完成
  - [x] 事项

  ## 待办事项
  - [ ] 事项

  ## 遇到的问题
  - 问题及解决方案

  ## 备注
  - 其他说明
  ```

### 3. 代码规范

- 修改 UI 前必须查看 `docs/03-UI设计规范.md`
- 修改功能行为前必须查看 `docs/01-需求文档.md`
- 使用原生 JavaScript（ES6+），不引入任何运行时依赖
- 注释用中文，代码变量名用英文

### 4. 需求变更

- 用户提出新需求或变更时，先更新 `docs/01-需求文档.md`
- 如涉及技术方案变化，同步更新 `docs/02-技术方案.md`
- 如涉及 UI 变化，同步更新 `docs/03-UI设计规范.md`
